점검 관점

속도 측정이 아니라 링크 동작 상태 확인

01

네트워크설비 점검의 핵심

  • 인터넷 속도측정 사이트 결과만으로 스위치 상태를 판단하지 않습니다.
  • 스위치 포트의 Link / Speed / Duplex / Error 상태 확인이 우선입니다.
  • 관리형 여부에 따라 점검 방식과 확인 가능한 범위가 달라집니다.
현장 표현으로는 “속도 측정”보다 링크 동작 속도 확인에 가깝습니다.
Speed100Mbps / 1Gbps / 10Gbps 등 링크 협상 속도
DuplexFull Duplex 여부 확인
ErrorCRC, FCS, Input/Output Error
Traffic주요 포트 송수신 상태

관리형 스위치에서 반드시 볼 항목

05
Link사용 포트가 Up 상태인지 확인
Speed장비·케이블 기준에 맞는 속도인지 확인
DuplexFull Duplex가 정상 기준
ErrorCRC/FCS/Input/Output Error 증가 여부 확인

확인 필요 또는 부적합 예시

  • 1Gbps 대상 포트가 100Mbps로 연결
  • Half Duplex로 표시
  • CRC/FCS Error가 계속 증가
  • 사용 중 포트가 Link Down
  • 업링크 포트의 트래픽·에러 이상
비관리형 점검

접속 불가 장비는 간접 확인이 현실적입니다

06
  • 장비 라벨·모델명으로 최대 지원속도 확인
  • 전원 LED와 포트별 Link/Act LED 확인
  • 운영 중인 케이블은 무리하게 분리하지 않음
  • 예비 포트 또는 점검 가능한 포트에 노트북 연결
  • PowerShell로 LinkSpeed와 FullDuplex 확인
비관리형은 포트별 에러 카운터와 내부 트래픽을 소프트웨어로 직접 볼 수 없습니다.
현장 노트북 점검

가장 현실적인 방법: PowerShell 확인

07
Get-NetAdapter | Where-Object {$_.Status -eq "Up"} | Select-Object Name, Status, LinkSpeed, FullDuplex
Name      Status  LinkSpeed  FullDuplex
----      ------  ---------  ----------
Ethernet  Up      1 Gbps     True

규격별 외관 경향

차폐 방식

  • UTP: 별도 차폐막 없는 일반 케이블
  • F/UTP 또는 FTP: 전체 포일 차폐
  • STP: 차폐 구조 케이블의 포괄 표현
  • S/FTP: 전체 차폐와 페어별 차폐 구조
차폐형은 케이블뿐 아니라 접지, 패치패널, 커넥터 구성도 함께 봐야 합니다.

마무리

최종 정리

14
  • 스위치 속도 점검은 인터넷 속도측정보다 링크 동작 상태 확인이 우선입니다.
  • 관리형은 Web·SSH·Console·SNMP로 전체 포트 상태를 확인합니다.
  • 비관리형은 외관, LED, 노트북 연결로 간접 확인합니다.
  • 케이블은 외피 CAT 표기와 실제 LinkSpeed를 함께 확인합니다.

현장 기본 명령어

Get-NetAdapter | Where-Object {$_.Status -eq "Up"} | Select-Object Name, Status, LinkSpeed, FullDuplex
노트북으로 점검할 때 가장 간단하고 현실적인 확인 방법입니다.
