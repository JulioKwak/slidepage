01

네트워크설비 점검의 핵심

  • 인터넷 속도측정 사이트 결과만으로 스위치 상태를 판단하지 않습니다.
  • 스위치 포트의 Link / Speed / Duplex / Error 상태 확인이 우선입니다.
  • 관리형 여부에 따라 점검 방식과 확인 가능한 범위가 달라집니다.

비관리형 점검

접속 불가 장비는 간접 확인이 현실적입니다

06
  • 장비 라벨·모델명으로 최대 지원속도 확인
  • 전원 LED와 포트별 Link/Act LED 확인
  • 운영 중인 케이블은 무리하게 분리하지 않음
  • 예비 포트 또는 점검 가능한 포트에 노트북 연결
  • PowerShell로 LinkSpeed와 FullDuplex 확인
비관리형은 포트별 에러 카운터와 내부 트래픽을 소프트웨어로 직접 볼 수 없습니다.

VLAN 기초

VLAN은 하나의 스위치를 여러 개의 가상망처럼 나누는 기능입니다

14

초보자용 설명

  • VLAN은 Virtual LAN의 약자입니다.
  • 물리적으로는 같은 스위치에 연결되어 있어도, 논리적으로는 서로 다른 네트워크처럼 분리합니다.
  • 예를 들어 업무망, CCTV망, 방문자망, 서버망을 같은 장비에서 분리해서 운영할 수 있습니다.
  • 분리된 VLAN 간 통신은 라우터, L3 스위치, 방화벽 정책을 통해 허용할 때만 가능합니다.
물리 스위치 1대
→
업무망 VLAN 10
CCTV VLAN 20
방문자 VLAN 30
서버 VLAN 40
VLAN은 물리 장비 1대를 논리적으로 여러 망처럼 분리하는 개념입니다.
VLAN 보안

VLAN 점검은 “분리되어야 할 망이 실제로 분리되어 있는지” 확인합니다

15

기본 확인 항목

점검 항목확인 내용
VLAN 목록VLAN ID, 명칭, 용도 확인
포트 배정각 포트가 어느 VLAN에 속하는지 확인
Access/Trunk단말 포트는 Access, 스위치 간 연결은 Trunk 여부 확인
미사용 포트Disable 또는 별도 격리 VLAN 처리 여부 확인
관리 VLAN장비 관리용 VLAN이 일반 사용자망과 분리되어 있는지 확인

보안상 특히 중요한 사항

  • 방문자망이 업무망·서버망으로 접근되지 않아야 합니다.
  • CCTV망, 출입통제망, 서버망은 필요 목적 외 통신을 제한해야 합니다.
  • Trunk 포트는 허용 VLAN을 필요한 것만 제한하는 것이 좋습니다.
  • 기본 VLAN 1에 중요 장비를 그대로 두는 구성은 피하는 것이 좋습니다.
  • VLAN 간 통신 허용 여부는 방화벽 또는 L3 장비 정책과 함께 확인해야 합니다.
VLAN은 망을 나누는 기능이지, 단독으로 완전한 보안장비가 되는 것은 아닙니다.
VPN 기초

VPN은 외부에서 내부망으로 안전하게 들어오는 암호화 통로입니다

16
VPN은 외부 사용자와 내부 시스템 사이에 암호화된 통신 터널을 만드는 개념입니다.

초보자용 설명

  • VPN은 Virtual Private Network의 약자입니다.
  • 인터넷을 지나가지만, 통신 내용은 암호화된 터널 안에서 이동합니다.
  • 외근자, 재택근무자, 지사, 협력업체가 내부 업무망에 접속할 때 사용합니다.
  • 대표적으로 SSL VPN, IPsec VPN, WireGuard 방식 등이 있습니다.
비유하면, 공용도로를 지나가지만 전용 방탄차 안에서 이동하는 것과 비슷합니다.
VPN 보안

VPN 점검은 “누가, 어떤 방식으로, 어디까지 접속하는지” 확인합니다

17

접속·계정 점검

점검 항목확인 내용
사용자 계정퇴사자, 미사용 계정, 공용 계정 존재 여부
인증 방식비밀번호만 사용하는지, MFA 적용 여부
접속 이력최근 접속자, 실패 로그인, 해외·이상 접속 여부
권한 범위접속 후 접근 가능한 내부 시스템 범위
외부 공개 상태VPN 장비가 불필요하게 넓게 노출되어 있지 않은지

장비·정책 점검

  • VPN 장비 펌웨어와 보안 패치가 최신인지 확인합니다.
  • 암호화 방식이 취약한 구형 방식으로 설정되어 있지 않은지 확인합니다.
  • Split Tunnel 허용 여부와 내부망 접근 범위를 확인합니다.
  • 인증서 만료일, 로그 보관, 세션 타임아웃 설정을 확인합니다.
  • 관리자 페이지는 일반 인터넷에 그대로 노출되지 않도록 제한하는 것이 좋습니다.
VPN 계정 하나가 탈취되면 내부망 진입로가 될 수 있으므로 계정 관리가 핵심입니다.
마무리

최종 정리

18
  • 스위치 속도 점검은 인터넷 속도측정보다 링크 동작 상태 확인이 우선입니다.
  • 관리형은 Web·SSH·Console·SNMP로 전체 포트 상태를 확인합니다.
  • 비관리형은 외관, LED, 노트북 연결로 간접 확인합니다.
  • 케이블은 외피 CAT 표기와 실제 LinkSpeed를 함께 확인합니다.

현장 기본 명령어

Get-NetAdapter | Where-Object {$_.Status -eq "Up"} | Select-Object Name, Status, LinkSpeed, FullDuplex
노트북으로 점검할 때 가장 간단하고 현실적인 확인 방법입니다.
